
# Generar SAS y actualizar `DATABASE SCOPED CREDENTIAL` 
# (PowerShell y az CLI)

Este documento contiene dos scripts de ejemplo para generar un SAS en un contenedor de Azure Blob y actualizar
la `DATABASE SCOPED CREDENTIAL` en la base de datos para que Extended Events pueda escribir en el contenedor.

Requisitos:
- Tener `Az` PowerShell module instalado (`Install-Module Az -Scope CurrentUser`) y el módulo `SqlServer` para `Invoke-Sqlcmd`.
- Alternativa: `az` CLI instalado para el ejemplo de Bash.
- Permisos: acceso para generar SAS en la cuenta de Storage y permisos para ejecutar ALTER DATABASE SCOPED CREDENTIAL en la DB.

---

## 1) PowerShell (Az) — generar SAS y actualizar la credencial en la base de datos

Uso: editar variables al inicio y ejecutar en un entorno con `Connect-AzAccount` ya autenticado.

```powershell
<# Parámetros (ajusta antes de ejecutar) #>
$subscriptionId = '<SUBSCRIPTION_ID>'
$resourceGroup = '<RESOURCE_GROUP>'
$storageAccount = '<STORAGE_ACCOUNT_NAME>'
$containerName = '<CONTAINER_NAME>'
$sasExpiryDays = 7

# Conexión a la DB (Azure SQL)
$sqlServer = '<your-server>.database.windows.net'
$database = '<your-database>'
$sqlUser = '<sql-admin-user>'
$sqlPassword = Read-Host -AsSecureString 'SQL admin password'

Import-Module Az.Storage -ErrorAction Stop
Import-Module SqlServer -ErrorAction Stop

# Seleccionar suscripción
Set-AzContext -Subscription $subscriptionId

# Obtener contexto de storage
$storageAccountObj = Get-AzStorageAccount -ResourceGroupName $resourceGroup -Name $storageAccount
$ctx = $storageAccountObj.Context

# Generar SAS para el contenedor (lectura/escritura/listado) — ajustar permisos según necesidad
$expiry = (Get-Date).ToUniversalTime().AddDays($sasExpiryDays)
$sasToken = New-AzStorageContainerSASToken -Name $containerName -Context $ctx -Permission rwl -ExpiryTime $expiry

# New-AzStorageContainerSASToken suele devolver el token SIN el prefijo '?'
$sasTokenClean = $sasToken.TrimStart('?')

# Construir la instrucción T-SQL para actualizar la credencial
$secret = $sasTokenClean
$tsql = "ALTER DATABASE SCOPED CREDENTIAL BlobStorageCredential WITH SECRET = '$secret';"

# Ejecutar el T-SQL en la base de datos
Invoke-Sqlcmd -ServerInstance $sqlServer -Database $database -Username $sqlUser -Password (ConvertFrom-SecureString $sqlPassword -AsPlainText) -Query $tsql

Write-Host "SAS generado y credencial actualizada. Expira: $expiry"

# Notas: en entornos productivos usa autenticación AAD/Managed Identity para aplicar cambios de forma más segura.
```

Notas PowerShell:
- `New-AzStorageContainerSASToken` crea SAS a nivel de contenedor. Ajusta `-Permission` a `rwld` o según el mínimo necesario.
- `Invoke-Sqlcmd` requiere el módulo `SqlServer`. Para Azure AD auth sustituye `-Username`/`-Password` por credenciales AAD.

## 2) az CLI (Bash) — generar SAS y actualizar la credencial (ejecución de T-SQL via sqlcmd)

Requisitos: `az` y `sqlcmd` instalados. Estar autenticado con `az login`.

```bash
# Variables
SUBSCRIPTION_ID="<SUBSCRIPTION_ID>"
RESOURCE_GROUP="<RESOURCE_GROUP>"
STORAGE_ACCOUNT="<STORAGE_ACCOUNT_NAME>"
CONTAINER_NAME="<CONTAINER_NAME>"
EXPIRY_DAYS=7

# Obtener clave de cuenta (necesaria para generar SAS con az CLI)
ACCOUNT_KEY=$(az storage account keys list -g $RESOURCE_GROUP -n $STORAGE_ACCOUNT --query "[0].value" -o tsv)

# Generar SAS para el contenedor (permisos: acw -> add/create/write)
EXPIRY=$(date -u -d "+$EXPIRY_DAYS days" '+%Y-%m-%dT%H:%MZ')
SAS_TOKEN=$(az storage container generate-sas --account-name $STORAGE_ACCOUNT --account-key $ACCOUNT_KEY --name $CONTAINER_NAME --permissions acw --expiry $EXPIRY --https-only --output tsv)

# Elimina el '?' si viene incluido
SAS_TOKEN_CLEAN=${SAS_TOKEN#\?}

# Construir y ejecutar comando sqlcmd para actualizar la credencial
SQLSERVER="<your-server>.database.windows.net"
DATABASE="<your-database>"
SQLUSER="<sql-admin-user>"

# Pide contraseña interactivamente
read -s -p "SQL password: " SQLPASS; echo

TSQL="ALTER DATABASE SCOPED CREDENTIAL BlobStorageCredential WITH SECRET = '${SAS_TOKEN_CLEAN}';"

echo "Updating credential..."
sqlcmd -S $SQLSERVER -d $DATABASE -U $SQLUSER -P $SQLPASS -Q "$TSQL"

echo "SAS generado y credencial actualizada. Expira: $EXPIRY"
```

Notas az CLI:
- Generar SAS con `az storage container generate-sas` usando la `account-key` es sencillo, pero manejar keys tiene implicaciones de seguridad.
- Para seguridad mejor usar User Delegation SAS con Azure AD y `az storage container generate-sas --auth-mode login --as-user` si es compatible.
